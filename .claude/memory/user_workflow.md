Daniel's interaction model with YETI has clear layers:

**Claude Code sessions (like this one):** Deep work — defining structure, bulk importing context, building features, debugging. This is where intensive thinking happens.

**Telegram:** Simple instructions, quick queries, photo capture (business cards, receipts). Not for intensive work sessions.

**Dashboard:** Surface important data, execute basic tasks (approve/reject actions, browse knowledge). A window into the system, not a workspace.

**Core goal:** Focus all information in one space. Free Daniel to focus on human skills and value — not get lost in small managerial tasks. YETI handles the information consolidation and routine management.

**Bootstrapping approach:** Define initial structure (profile, company, projects) in a Claude session, then bulk-import context from existing markdown files (past Claude sessions). After that, interactive additions via any channel.
